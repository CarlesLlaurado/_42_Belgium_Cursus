# ft_printf
 
## Description
 
`ft_printf` is a reimplementation of the C standard library function `printf()`. The goal is to understand variadic functions in C — functions that accept a variable number of arguments — and to build a clean, extensible library that handles formatted output.
 
The library is compiled as a static archive (`libftprintf.a`) and supports the following conversion specifiers:
 
| Specifier | Output |
|-----------|--------|
| `%c` | Single character |
| `%s` | String |
| `%p` | Pointer address in hexadecimal (`0x` prefix) |
| `%d` | Signed decimal integer |
| `%i` | Signed integer in base 10 |
| `%u` | Unsigned decimal integer |
| `%x` | Unsigned hexadecimal (lowercase) |
| `%X` | Unsigned hexadecimal (uppercase) |
| `%%` | Literal percent sign |
 
## Instructions
 
### Compilation
 
```bash
make
```
 
This produces `libftprintf.a` at the root of the repository.
 
### Usage
 
Include the header and link the library when compiling your own program:
 
```bash
cc main.c -L. -lftprintf -o my_program
```
 
```c
#include "ft_printf.h"
 
int main(void)
{
    ft_printf("Hello, %s! Answer is %d.\n", "world", 42);
    return (0);
}
```

## Algorithm and Data Structure
 
### Overall design
 
`ft_printf` scans the format string character by character. When it encounters `%` followed by a valid specifier, it delegates to a dedicated helper function. Otherwise it writes the character directly to stdout. The return value accumulates the total number of characters written, mirroring the behavior of the real `printf()`.
 
```
ft_printf() → scan format string → '%' found → ft_handle_conversion() → helper function
                                 → plain char → write() directly
```
 
### Variadic argument handling
 
The C standard `<stdarg.h>` macros are used:
- `va_start` initializes the argument list after the last named parameter.
- `va_arg` pulls the next argument with the expected type.
- `va_end` cleans up.
A pointer to `va_list` (`va_list *args`) is passed to `ft_handle_conversion` so that consuming an argument inside the helper advances the shared cursor correctly.
 
### Number printing (recursive digit extraction)
 
All numeric helpers (`ft_print_int`, `ft_print_uint`, `ft_print_hex`) use the same recursive strategy:
 
1. Divide by the base (10 or 16) recursively until a single digit remains.
2. On the way back out of the recursion, print each digit from most significant to least significant.
This avoids the need to allocate a buffer to reverse the digits. The only edge case is `INT_MIN` (`-2147483648`), which cannot be negated safely in a 32-bit integer — it is handled by printing the string literal directly before any arithmetic is attempted.
 
### Pointer printing
 
Pointers require `unsigned long` (64 bits) instead of `unsigned int` (32 bits) because memory addresses on 64-bit systems occupy 8 bytes. A separate helper `ft_print_hex_ptr` handles this to avoid truncating the upper 32 bits of an address.
 
### Static library
 
The library is built with `ar rcs`, as required by the subject. The flags mean: `r` (insert/replace members), `c` (create archive if absent), `s` (write symbol index). `libtool` is not used.
